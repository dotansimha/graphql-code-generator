## Guide

Most GraphQL API resolvers remains untyped or wrongly typed which leads to multiple issues:

- resolvers not compliant to the Schema definition

- unhandled scenarios

- typos in the resolvers' function type signature

<br/>

For this reason, GraphQL Code Generator provides `@graphql-codegen/typescript-resolvers` that helps automating the generation of resolvers' typings.

<br/>

Just a few configuration steps are required to get the resolvers types generated:

**1. Move your GraphQL Schema declaration in dedicated `.graphql` files**

```graphql
#schema.graphql

type Author {
  id: Int!
  firstName: String!
  lastName: String!
  posts(findTitle: String): [Post]
}

type Post {
  id: Int!
  title: String!
  author: Author
}

type Query {
  posts: [Post]
}
```

**2. Install the `@graphql-codegen/typescript-resolvers` plugin**

<PackageInstall packages={["@graphql-codegen/typescript-resolvers", "@graphql-codegen/typescript"]} />

<br />

**3. Configure the plugin**

Create or update your `codegen.yaml` file as follows:


<CodeTabs namespace={`guides-apollo-server-graphql-yoga-codetabs`}>

<CodeTab label="Apollo Server">

```yaml
schema: schema.graphql
generates:
  ./resolvers-types.ts:
    config:
      useIndexSignature: true
    plugins:
      - typescript
      - typescript-resolvers
```

</CodeTab>

<CodeTab label="GraphQL Yoga">

```yaml
schema: schema.graphql
generates:
  ./resolvers-types.ts:
    plugins:
      - typescript
      - typescript-resolvers
```

</CodeTab>

</CodeTabs>


<br />

**4. Run the codegen and update your code**

Assuming that, as recommended, your `package.json` has the following script:

```json
{
  "scripts": {
    "generate": "graphql-codegen",
  }
}
```

<br />

Running the following will generate the `graphql/generated.tsx` file.

<PackageRun scripts={['generate']} />

<br />

We can now write/migrate our GraphQL API implementation as follows:


<CodeTabs namespace={`guides-apollo-server-graphql-yoga-codetabs`}>

<CodeTab label="Apollo Server">

```ts
import { ApolloServer } from "apollo-server";
import { loadDocuments } from "@graphql-tools/load";
import { GraphQLFileLoader } from "@graphql-tools/graphql-file-loader";

import { Resolvers } from './resolvers-types';


const typeDefs = await loadDocuments('./schema.graphql', {
    file,
    loaders: [
        new GraphQLFileLoader()
    ]
})


const resolvers: Resolvers = {
  Query: {
    // typed resolvers
  }
}

const server = new ApolloServer({ typeDefs, resolvers });

// The `listen` method launches a web server.
server.listen().then(({ url }) => {
  console.log(`ðŸš€  Server ready at ${url}`);
});
```

</CodeTab>

<CodeTab label="GraphQL Yoga">

```ts
import { createServer } from '@graphql-yoga/node'
import { loadDocuments } from "@graphql-tools/load";
import { GraphQLFileLoader } from "@graphql-tools/graphql-file-loader";

import { Resolvers } from './resolvers-types';

const typeDefs = await loadDocuments('./schema.graphql', {
    file,
    loaders: [
        new GraphQLFileLoader()
    ]
})

const resolvers: Resolvers = {
  Query: {
    // typed resolvers
  }
}

const server = createServer({
  typeDefs,
  resolvers,
})

server.start()
```

</CodeTab>

</CodeTabs>
